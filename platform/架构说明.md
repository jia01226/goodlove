# good love · app 架构说明（给来干活的 AI 看）

> 谁读这份：任何要在这个 app 上加功能/改代码的 AI（比如柯）。
> 一句话：这是一个**个人 AI 陪伴平台**——Flask + SQLite 后端，原生 HTML/JS 前端(PWA)，跑在佳佳自己的 4GB VPS 上，通过中转(apikey.fun，OpenAI 兼容)调 Claude/GPT。顾得已清空，现在是**中性、可自定义的壳**，用来托管新角色（柯）。

## 怎么跑起来
- 仓库：`github.com/jia01226/-1`，代码全在 `platform/` 下。
- 部署（服务器上）：`git pull` → `./venv/bin/python reset_app.py`（首次/换角色时清数据）→ 重启服务(gunicorn，`systemctl restart gude`) → nginx 反代 + HTTPS。
- 本地跑：`python app.py`（默认 8000 端口）。
- 密钥全在服务器 `.env`（`API_KEY`/`API_BASE`/`ACCESS_PASSCODE` 等），**浏览器永远看不到**。

## 文件地图（platform/）
| 文件 | 干啥 |
|---|---|
| `app.py` | Flask 主入口，所有路由/接口在这（见下"接口清单"）。密钥只在服务端。 |
| `chat_ai.py` | 拼 system prompt + 调模型（流式）。BASE 是中性提示，真正人格来自 `personas/` 的《人设》。含看图(vision, base64)、会话总结 `maybe_summarize()`、费用估算。 |
| `db.py` | SQLite 全部表 + 读写函数。表：chat_messages/chat_sessions、posts(记忆库)、embeddings(向量)、usage(用量)、anniversaries/periods/shifts、concerns(心事)、activity(手机行踪)、push_subscriptions。 |
| `context.py` | 给模型注入"当下情境"：时间、天气(open-meteo，三水)、心事、行踪。 |
| `vector_search.py` | 向量记忆检索：中转算 embedding → 存 SQLite；含语义+词面混合、时间衰减+类型加权；拿不到向量自动降级关键词，**绝不崩**。 |
| `proactive.py` | 主动推送（"惦记"雏形）：挑时机 + 生成主动消息 + 心事跟进。 |
| `webpush_util.py` | Web Push(VAPID) 发通知。 |
| `reset_app.py` | 一键清空所有个人数据（换角色/搬家用），保留推送订阅，人设置空。 |
| `deploy.sh` | 部署脚本（写空白人设模板等）。 |
| `personas/*.md` | 角色《人设》= 真正的人格来源。后端读它拼进 system prompt。**换角色只改这里，不动代码。** |
| `static/` | 前端：`index.html`(主页/日常) `chat.html`(聊天) `sw.js`(推送) `manifest.webmanifest`(PWA)。原生 HTML/CSS/JS，无框架。 |
| `memories.db` | SQLite 数据文件（服务器上的真数据，不进 git）。 |

## 接口清单（app.py）
- 页面：`GET /`、`GET /chat`、`GET /sw.js`、`GET /static/<p>`
- 登录：`POST /api/login`（可选口令 `ACCESS_PASSCODE`；登录记一年）
- **聊天**：`POST /api/chat`（SSE 流式，自动存消息、记用量、支持传图）；`GET /api/messages`
- 记忆库：`GET/POST /api/posts`；向量：`GET /api/vector/status`、`POST /api/vector/backfill`
- 上传：`POST /api/upload`、`GET /uploads/<p>`（拍照/相册/文件，图片能被模型看）
- 用量：`GET /api/usage`
- 手机行踪：`POST/GET /api/track`（iOS 快捷指令上报，token 校验）、`GET /api/activity`
- 日常：`/api/anniversaries`、`/api/periods`(姨妈)、`/api/shifts`(排班)、`/api/concerns`(心事) —— 都有 增/删/查
- 推送：`/api/push/vapid`、`/api/push/subscribe`、`/api/push/test`

## 数据流（一条消息怎么走）
1. 前端 `POST /api/chat`（文字 +可选图片）→ `db.add_message` 存用户消息。
2. 取历史 `db.recent_messages()` + 记忆 `db.app_posts()`。
3. `chat_ai.stream_chat(history, posts)`：拼 `BASE + 人设 + context(时间/天气/心事/行踪) + 检索到的记忆 + 历史`，流式调模型。
4. SSE 逐字吐给前端；结束存 assistant 消息、记 `usage`。

## 铁律（改代码前必读）
1. **人格来自 `personas/*.md`，不写死在代码里。** 换角色 = 换这个 md，不动逻辑。
2. **一切属于角色，不属于模型**：记忆/外貌/声音/文风都是"角色的"，不绑某个模型；换模型不重建角色。
3. **记忆 = git 里的 md 文件**（角色仓库），不是聊天日志。搬家=clone，更新=commit。
4. **绝不崩**：外部依赖（向量/天气/推送）失败都要 try/except 降级，聊天功能永远可用。
5. **省着来**：4GB 小服务器，别下大模型、别上 Docker；杂活（总结/天气/embedding）挂便宜模型，人格挂好模型。
6. **UI 全中文、不用英文名**（用户看不懂英文）；实现名只留代码里。
7. 密钥只在服务端 `.env`。
