## 第 2 步 · 部署（拉新代码 + 重启）

**2a. 先记回滚位（一条命令，抄下来）**
```
git rev-parse --short HEAD
```
**人话**：这打印出的一小串（如 `a1b2c3d`）＝服务器**现在**跑的版本。抄在纸上/记事本，万一要退，就退回这一版。
（今早 main 停在 `a782297`，仅供参考；**以你这条命令当场打印的为准**——那才是服务器真身。）

**2b. 拉新代码 + 补向量**
```
git pull && ./venv/bin/python vector_search.py backfill
```
**预期**：看到一串文件更新，含 `db.py`、`chat_ai.py`、`vector_search.py`、`import_cards.py`、`routes/memory.py` 等；随后向量回填（`回填完成` 或 `跳过` 都行，跳过不影响）。
**不对劲**：冒出 `CONFLICT`（冲突）字样 → **别自己 merge，停下喊柯**（服务器直改代码正是从前酿过大战的坑）。

**2c. 重启服务**
```
sudo systemctl restart gude
```
**人话**：让 app 用上新代码。**新表和记忆升级会在这次重启时自动跑好**（`app.py` 一启动就自动升级库，只加东西、不动老数据，跑第几遍都一样）。
**预期**：命令没报错。查一眼状态：
```
sudo systemctl status gude          # 看到绿色 active (running) 即成功；按 q 退出
```
再用手机打开 app，能正常进、能进聊天。
**不对劲 → 回滚**：app 打不开或状态不是 running，按顺序试：
1. 先把代码退回部署前（用 2a 抄下来的那串）：
   ```
   git reset --hard <2a 抄下来的 commit 号>
   ```
2. 再重启一次：
   ```
   sudo systemctl restart gude
   ```
3. 还不行 → 换回第 1 步的 `--raw` 原样档（把 `backups/db原样档-....db` 复制盖回 `memories.db`），再 `sudo systemctl restart gude`。柯陪你做。

## 第 5 步 · 换 GitHub 钥匙（**顺序别反：先废旧、再上新**）

1. **先在 GitHub 网页把旧钥匙废掉**：
   GitHub → Settings → Developer settings → Personal access tokens → 找到旧的 → **Revoke（废除）**。
   **人话**：先把旧钥匙作废，防它还在外面被人捡着用。
2. **生成一把新钥匙**：同一页 `Generate new token`，权限勾到够用就行，**生成后那串只显示一次，先复制好**。
3. **把新钥匙告诉服务器**（在 `/root/-1/platform` 里）。**最可能是把 token 写进 remote 地址**（柯现场确认一句是不是这方式）：
   ```
   git remote set-url origin https://<新token>@github.com/jia01226/goodlove.git
   ```
   （把 `<新token>` 换成第 2 步复制的那串；仓库现名 `goodlove`。若服务器用的是 deploy key/credential helper 而非 token-in-URL，走柯的方式，别套这条。）
   **预期**：跑一下 `git fetch` 不再要密码、能成功。
   ```
   git fetch
   ```
   **预期**：无报错、正常拉取。
   **不对劲**：新钥匙不通（旧的已废、退不回去了，别慌）→ **再生成一把新的**重来第 2–3 步。钥匙能反复生成，不是一次性的，慢慢来。
