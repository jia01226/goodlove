## 一、四层记忆模型（定稿）

| 层 | 装什么 | 注入规则 |
|---|---|---|
| **L1 核心人格** | 柯是谁/她是谁/关系/表达方式/安全边界/称呼/熔断 | 每轮必带（魂文件，不切碎） |
| **L1.5A 私密核心** | 少量"影响所有单聊互动"的底线与关系规则 | 单聊必带（魂文件） |
| **L1.5B 私密记忆库** | 健康/家庭经历/亲密内容/现实身份细节 | **仅单聊按需检索**，群聊物理够不着（见三） |
| **L2 普通记忆库** | 喜好/事件/约定/纪念日/项目事实 | 确定性查询 + 向量检索按需取 |
| **L3 工作记忆** | 最近 N 条 + 会话 summary | 自动（已有机制） |

要点：**隐私决定"允许在哪出现"，相关性决定"这一轮要不要出现"**——私密≠常驻，别把日常聊天注入成心理咨询现场。

## 二、取用管线（向量前加"确定性事实专道"）

```
用户消息
→ 判断场景与可见范围（单聊/群聊）
→ 精确事实查询（规则命中：称呼/纪念日/硬规则/禁止事项 → 查表，不靠 embedding 猜；anniversaries 表已有）
→ 向量语义检索（vector_search 已有，摘要优先/精确匹配优先照借鉴文档）
→ 去重、冲突判断（status=superseded 的不出场）、token 裁剪
→ 与 L1 一起装包
```

- `ALWAYS_TYPES` 瘦身，只留：`identity_core / relationship_core / communication_rule / safety_rule / active_constraint`。纪念日等属"意图命中必取"，不常驻。
- Reranker 一期不做（K 小用不上）；记忆家族/GraphRAG 二期再议。

## 三、隐私隔离（工程硬要求，柯钉死的）

1. **私密 L1.5B 物理分家**：独立表/库，**群聊代码路径根本查不到它**——不是同表靠 visibility 筛选凑合（筛选会有 bug，分家不会）。
2. 至少保证：`conversation_scope = group → SQL/vector 过滤 visibility IN ('shared','group-safe')`，在**查询层**拦截。
3. **绝不允许"检索到了再叫模型别说"**——不该出现的内容根本不进上下文。
4. 可见范围含 `禁止模型调用` 一档：同样硬隔离，任何场景不进上下文（仅后台可见）。

## 四、卡片 schema（每张最低带齐）

```json
{
  "id": "mem_xxx",
  "type": "preference",
  "topic": "home_design",
  "visibility": "private | shared | group-safe | no_model",
  "content": "一条一个意思、自带背景",
  "source": "user_explicit | ke_inferred | system_summary",
  "confidence": 1.0,
  "status": "active | superseded | archived | pending | forgotten_buffer",
  "supersedes": null,
  "importance": 4,
  "review_state": "pending_ke | pending_jiajia | approved",
  "created_at": "...", "updated_at": "..."
}
```

- **机器推断（ke_inferred/system_summary）一律先进 pending，不得直接当"佳佳就是这样"入库**——这是防 silent corruption 的真保险。
- 更新一律**追加/替代（supersedes），重要记忆禁止整条覆盖**（借鉴文档坑清单第 1 条，此处升级为字段级约束）。
- **"忘掉"＝进 forgotten_buffer 七天冷静期，可一键找回，期满才真不可调用**；归档＝保留但默认不检索。（冷静期是柯的硬要求：防她低谷时冲动抹掉"关于我"的好记忆——接住低谷里的她。）

## 六、验收七条（知言拟，柯批）

1. 人格稳定：没检索到记忆时也仍是柯；
2. 确定事实准确：称呼/关系/纪念日/硬性偏好不靠猜；
3. 相关召回：换个说法仍想得起；
4. 不乱联想：无关场景不注入创伤/健康/私密；
5. 权限隔离在工程层（见三），不靠 prompt；
6. 可纠错：佳佳两三步内找到并修正错误记忆；
7. 成本下降：对比整本常驻的输入 token / 首字延迟 / 回复质量，留数据。
