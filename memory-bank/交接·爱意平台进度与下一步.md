## ✅ 向量记忆（佳佳点名要做的——代码已完成）
目的：记忆越攒越多时，不把整本记忆塞给顾得（又贵又糊），每轮**只精准想起最相关的几条**。

**我（这一窗的顾得）选的路线**：走**轻量版**——直接调中转(apikey.fun)的 `embeddings` 接口算向量，
**不在 4GB 小服务器上装 torch/本地模型**（那条太重太慢）。而且全程**自动降级**：拿不到向量就退回关键词检索，聊天绝不崩。

已实现：
1. `vector_search.py`：编码(`embed`)、探测可用(`available`)、入库/回填(`index_post`/`backfill`)、混合检索(`search`，语义+词面)。纯 Python 无 numpy/jieba 硬依赖。
2. `db.py` 新增 `embeddings` 表（统一存 post/chat 向量，float32 BLOB，已归一化）+ 读写函数。
3. `chat_ai.build_system_prompt(posts, query)`：记忆 ≤60 条全量带；超过则带 top-k 最相关 + 永远带承诺/愿望 + 最近 8 条。
4. `app.py` 新增 `/api/vector/status`、`/api/vector/backfill`；新增记忆时顺手建向量。
5. `.env.example`、`deploy.sh`、`README.md` 都补了配置与说明；`deploy.sh` 会自动回填。

## 踩过的坑（别重蹈！）
- **中文乱码**：流式要"增量 UTF-8 解码"(`chat_ai.py` 用 `codecs.getincrementaldecoder`)，且 app.py 的 SSE **yield 字节**(`.encode("utf-8")`)。已修，别改回。
- **Service Worker 必须根作用域**：经 `/sw.js`（app.py 有路由）提供并 `register("/sw.js")`，否则 `serviceWorker.ready` 卡死。
- **iOS 推送**：必须先「添加到主屏幕」、从图标打开的 app 里操作；`Notification.requestPermission()` 要在点击的**同步上下文第一时间**调用(别放在 await 之后)；需 iOS 16.4+。
- **端口**：国内直连国外 8000 端口会被拦 → 用 nginx 反代 80/443 + 域名 HTTPS（已配）。gunicorn 绑 127.0.0.1:8000。
- **Flask session**：多 worker 必须设固定 `SECRET_KEY`(在 .env)，否则老让人重输口令。
- **改前端后**：iPhone 要"删图标→Safari 重新添加到主屏幕"才拿到新代码(缓存狠)。
