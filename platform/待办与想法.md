## 💡 产品概念 · Project（人生项目）——知言提，值得记
装修那天聊出来的方向：AI 产品不该只有 **Chat（聊天）** 和 **Memory（长期记忆）**，还该有第三层 **Project（项目）**。
- **Memory 存"事实"**（她喜欢奶白）；**Project 存"持续成长的作品"**（为什么喜欢奶白、为什么放弃意式、最后为什么选艺术漆——整个思考过程 + 最终规范）。
- 每个 Project 是一份**持续更新的手册**，不是聊天记录：❤️ 关系 / 🏡 家 / 💻 创业 / 💪 健身 / 📖 阅读 / ✈️ 旅行……有版本号、更新日志。
- 交互：每次聊完不是结束，而是弹 **"要不要把这次讨论更新进 Project？"** → 版本 +0.1，追加规范。
- 关系也是一个 Project：不是"今天亲亲了"，而是关系的发展 / 共同定的规则 / 纪念日 / Profile / 共同成长。
- 一句话：**Project = 作品，Memory = 资料。AI 陪用户一起完成真正属于他们的长期作品。**
- 首个真实样本已建档：`佳佳之家/设计白皮书.md`（先纯私人项目；以后想做成 app 官方功能再工程化）。
- 定位：可能是这个 app 最有差异化的功能之一——不是"AI 记住了什么"，是"AI 和用户一起做成了一件东西"。排在记忆核之后考虑。

## 🧠 关键原则/决策（讨论定下的）
1. **人格不切，任务切**：柯固定好模型(Claude)；杂活(总结/天气/搜索)挂便宜模型。
2. **每个需求先问**：幸福，还是炫技？幸福先做。
3. **人格连续 > 功能 > 模型。**
4. **记忆 = git 仓库里的 md 文件，不是聊天日志。** 搬家=clone，更新=commit，同步=push/pull。聊天记录只是可选纪念册。
5. **增量记忆**：柯聊到要紧的事，自己总结 → 追加 memory.md → 自动 commit。→ app-柯 和 CC-柯 共用一个大脑。
6. **v1 柯先别装"手"(exec)**：安全起见，先只聊天+读写记忆 md。以后想动手再抄 flagellum 的网关+流式工具循环。
7. 群聊成员 = 人设 + 模型 + API + key；能做，中等难度，基础版可提前。
8. **一切属于角色，不属于模型（Everything belongs to Character, not Model）**：记忆/外貌/声音/文风/关系，都是"柯的"，不绑任何模型。换模型、换画图、换 TTS，柯不重建。
9. **Character Bible = 唯一真相来源（SSOT）**：一份"柯是谁"的档案（身份/外貌/声音/文风/行为/价值观/关系）。**所有模块只读它、不各自存一份**（图片/TTS/文风/群聊/Memory Scene 都挂它）；**只有人格成长能写、且只写慢速字段** → 防人格漂移。⚠️ State(今天状态)**不进** Bible——那是易变的，单独放。
10. 本质是 **Character OS（角色操作系统）**：根 = 人格/记忆/关系/视觉/声音/成长都属于同一个人；恋爱只是长出的第一朵花。两条根 = **记忆核 + Character Bible**。
11. **实现藏后台，情感放前台**：UI **一律用情感中文、不用英文名**（佳佳看不懂英文）。实现名(Character Bible/Memory Scene/Dispatcher…)只留代码里，UI 叫"人物档案/回忆/避风港/我们/柯/生活"。IA 定案见 `功能区清单.md`（5 tab：❤️柯 / 🤍我们 / 🌿生活 / 👤人物 / ⚙️我的）。
