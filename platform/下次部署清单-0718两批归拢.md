预期：两行 ✅（原样档 + 备份 N 张表）。没看到就停。

**第 1 步·记回滚位 + 拉新代码 + 重启**（一整行）
```
cd /root/-1/platform && git rev-parse --short HEAD && git pull && sudo systemctl restart gude && sleep 3 && systemctl is-active gude
```
预期：先打印回滚版本号（截图存好）→ 一串文件更新 → 最后一行 `active`。
> 若冒 `CONFLICT`：多半是服务器上又有手改（像上次 protocol）——停下截图喊小克，别自己合。

**第 2 步·改主动消息 cron**（部署当次一并做）
```
crontab -e
```
把 proactive 的 8 行定点改成一行巡逻（UTC 写法）：`*/20 0-15 * * *`（北京 8:00–23:40 每20分钟一趟）；深夜守夜档照旧保留。
> ⚠️ 改完**逐条折回北京时间念一遍对账**（问药钟 07:02 误响就是 UTC 没换算栽的）。顺手把 crontab 里所有闹钟做一次全量时区审计——睡反了的当场扶正。
