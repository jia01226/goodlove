## 知言定案 · IA（UI 分区，已采纳）
体验原则：**实现藏后台，情感放前台。** 用户脑子里只有"柯 / 我 / 我们"，不该看到 Character Bible / Memory Scene / Dispatcher 这种实现名。**UI 一律用情感中文，不用英文名（佳佳明确要求：看不懂英文）。**

底部 5 个 Tab（别超过 5）：
1. **❤️ 柯（首页）**：打开就到这。只有柯 + 聊天(发图/语音/群聊)；柯主动找你也在这。**首页别堆天气/姨妈/待办**——聊天就是聊天。
2. **🤍 我们（灵魂）**：回忆、幸福数据库、避风港、纪念日、回忆胶片、照片、相册、时令、日记。**做成"像相册"不是列表**(类 Apple Photos：去年今天📷新疆…)。
3. **🌿 生活**：现实全放这——天气、姨妈、轮班、心事/待办、味觉地图、生活足迹。**把现实从恋爱里挪开、不打扰**。(解决"日常页太挤")
4. **👤 人物**：柯档案、相册、声音、成长、群聊成员(以后知言/小克也在)。人物馆。
5. **⚙️ 我的**：设置、模型、缓存、数据(查/改/删/导出)、会员、API。

UI 命名（后台实现名 → 前台情感名）：
- Character Bible → **人物档案 / 关于柯**
- Memory Scene → **回忆**
- Safe Events → **避风港**
- 幸福数据库 → 保留(有温度)
- Dispatcher 等实现名 → 不进 UI
