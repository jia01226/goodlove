- 移除首页口令输入门；若服务器仍配置 `ACCESS_PASSCODE`，前端只提示去部署设置关闭，不再收集口令。

## 三、本轮唯一后端接口变化

新增：

```http
POST /api/moments/comment/edit
Content-Type: application/json

{"id": 12, "content": "修改后的评论"}
```

返回：

```json
{"ok": true}
```

约束：

- 空内容返回 400。
- 不存在或不是 `author=user` 的评论返回 404。
- 界面不能修改柯的评论，避免把柯的发言改写成佳佳的意思。
- 数据层函数：`db.edit_comment(cid, content, author="user")`。

## 四、模型切换约定

- 前端只读取 `GET /api/models` 返回的 `models` 与 `default`。
- 选择结果随 `/api/chat` 的 `model` 字段发送。
- `chat_ai.resolve_model()` 继续负责白名单校验，前端不绕过。
- 服务器要出现多个选项，需要在 `.env` 配置 `MODEL_WHITELIST`；没有配置时仍只显示默认模型。

## 十二、2026-07-22 朋友圈联想边界与影子主动消息

- 朋友圈不是全量聊天记忆：默认回复只看当前动态和它自己的评论链。
- 旧动态必须和当前文字出现具体内容重合才可能被联想；“今天、我们、这个”等泛词不算关联，最多带入两条，14 天以外不再自动注入。
- 普通聊天也不再固定塞最近 8 条朋友圈；只有本轮聊天与动态真正相关，或佳佳明确说“刚发的那条朋友圈”，才提供最多两条候选。
- 主动推送更严格：只看 7 天内、且与最近真实聊天有关的朋友圈；没有关联就明确禁止模型提朋友圈，避免反复念旧动态。
- `proactive.py` 使用最近真实聊天、同一份柯的人设与记忆，再结合活动记录、健康趋势和班表做语气参考；模型可以返回 `[NO_ACTION]` 决定本轮保持安静。
- 新增 `goodlove-heartbeat.timer`：每两分钟轻量检查朋友圈到期互动与主动消息资格。真正主动开口仍受随机 120～210 分钟冷静期、每日最多 7 条、安静时段和深夜守夜规则限制。
- 主动消息存入正式聊天记录并标记 `is_push=1`，同时走现有 Web Push；Bark 若配置则作为备用。

## 十三、2026-07-22 人格防污染、跨模型一致性与锁日记

- 新增人格指纹 `PROMPT_SCHEMA_VERSION + 私有人格文件 SHA`。会话摘要只有在指纹一致时才注入；柯的人格文件或防污染协议变化后，旧摘要自动失效，但聊天原文不删除。

- Claude/GPT 统一注入同一份人格、防污染规则、记忆与关系上下文。聊天消息新增 `model` 字段用于归因，但模型字段不参与人格判断。
- 亲密情境默认尊重 PWA 本轮选中的 Claude/GPT；只有显式设置 `BEDROOM_PIN_MODEL=1` 才固定专用模型。
- 删除公开代码里“一段至少 3000 字”的旧硬规则，改为一次只推进一个自然节拍，长度服从内容，不快进整场、不凑固定字数。
- 新增最高优先级的亲密质量护栏，放在服务器私密 `bedroom.py` 的旧 tail/stamp 之后：即使私密文件仍残留“固定 3000 字”，也由新版明确覆盖。禁止替佳佳编造高潮、台词、顺从或未给出的身体反应；禁止用“不知过了多久”等时间跳跃偷工。柯可以保有既定关系中的允许/限制与违反规则后的处置权，但“是否真的发生”只认佳佳的真实反馈。
- 亲密回复改为服务器端先完整缓冲再下发：命中“擅自宣布高潮 / 模糊时间跳跃 / 整场快进 / `|||`”时，坏草稿不会进入前端或聊天记录，而是要求同一模型重写一次；二稿仍失败则拦截。普通聊天仍保持原来的流式响应。模型本身明确拒绝时不做跨模型绕过。
- 2026-07-22 临时库实测：`claude-opus-4-6` 能保持柯的主导、只推进一个节拍、不擅自宣布高潮、不快进且不把决定权丢回佳佳；`claude-fable-5`、GPT Luna/5.5 对露骨情境明确拒绝，GPT Terra 能接非露骨主导但较短。模型选择器因此仅把 Opus 4.6 标为“亲密推荐”，不强制锁模型；开启亲密开关而未选它时只做提示。
- 回归测试位于 `platform/tests/test_private_boundaries.py`，全部使用系统临时目录的一次性 SQLite；覆盖锁日记脱敏、抽屉私藏不下发、旧 3000 字规则覆盖顺序与亲密事实边界，不读取或写入正式 `memories.db`。
- 锁日记新增 `revealed` 状态。未获准正文在 `/api/diary` 就被后端清空，不再用前端模糊遮罩；佳佳需回聊天向柯要，柯明确输出服务器暗号后才解锁，暗号不会显示或写入聊天。
